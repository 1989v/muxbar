## 1. Overview

macOS 메뉴바에 상주하며 tmux 세션을 실시간으로 나열·모니터링·제어하는 네이티브 앱.
부가로 `caffeinate -dims` 를 **tmux 세션으로** 띄워 토글하는 "Keep Awake" 기능을 제공한다.

### 1.1 Problem

- tmux 세션을 여러 개 굴리는 개발자는 터미널로 `tmux ls` → `tmux a -t …` 를 반복한다.
- 백그라운드에서 돌고 있는 세션의 **현재 상태(무슨 로그가 찍히는지)** 를 빠르게 엿볼 수단이 없다.
- `caffeinate` 토글 메뉴바 앱은 많지만, **세션 관리와 통합된** 도구는 없다.

### 1.2 Goals (v0.1 MVP)

| # | Goal | 측정 |
|---|---|---|
| G1 | 메뉴바 클릭 한 번으로 모든 tmux 세션을 **리스트·상태** 확인 | 세션 목록 ≤200ms 에 갱신 |
| G2 | 세션 단위 **Attach / Kill** | 5개 터미널(Terminal/iTerm2/Warp/Alacritty/Kitty) 자동 감지·선택 |
| G3 | **라이브 프리뷰 (L3)** — 최근 활동/출력 실시간 스트리밍 | 30+ FPS, CPU <3% idle |
| G4 | **Keep Awake 토글** — `caffeinate -dims` 을 `_muxbar-awake` tmux 세션으로 | 메뉴 1클릭 ON/OFF, 플래그 설정 가능 |
| G5 | **전역 단축키** (⌘⇧T / ⌘⇧1-9 / ⌘⇧A) | 즉시 반응 |
| G6 | **세션 템플릿** (Dev/WebDev/Monitoring/SSH/Docker) | 1클릭 생성 |
| G7 | **시스템 알림** (세션 idle > 30m, 비정상 종료) | UserNotification 센터 통합 |
| G8 | **Login Item** 지원 (기본 OFF) | SMAppService.mainApp |
| G9 | GitHub Releases + Homebrew Tap 배포 | `brew install --cask 1989v/tap/muxbar` |

### 1.3 Non-Goals (v0.1 out of scope)

- 크로스 플랫폼 (Linux / Windows)
- tmux 원격(SSH) 세션 접속 관리
- 세션 녹화/재생
- Mac App Store 배포 (샌드박스로 tmux Process 실행 불가)
- 커스텀 tmux 테마 편집기
- 원격 공유/협업

### 2.3 Data Flow (Session Update)

```
tmux -C stdout
   ↓ (line by line)
ControlProtocol.parse() ──► Event
   ↓
ControlClient dispatches to SessionStore
   ↓
SessionStore updates @Published sessions
   ↓
SwiftUI MenuBarExtra rebuilds diff
```

**Key property**: tmux 이벤트는 **push** 이므로 폴링 없음. 초기 sync 만 `list-sessions` 호출.

### 2.4 Data Flow (Live Preview)

```
user hovers session "dev"
   ↓
SessionPreview opens popover
   ↓
request `capture-pane -pt dev -J -e` (snapshot)
   ↓
SwiftTerm.feed(snapshot)
   ↓
subscribe to %output for that pane
   ↓
each %output → SwiftTerm.feed(bytes) → NSAttributedString diff
```

## 3. Core Components

### 3.1 TmuxKit

#### 3.1.1 ControlClient

**부트스트랩 순서**:
1. `tmux has-session 2>/dev/null` 로 서버 존재 여부 확인
2. 없으면 `tmux new-session -d -s _muxbar-ctl` 로 detached 관리용 세션 기동 (서버 존재 보장 목적)
3. `tmux -C attach -t _muxbar-ctl` 프로세스 스폰 → 제어 모드 진입

**특성**:
- 제어 모드 클라이언트는 attach 대상과 무관하게 **서버의 모든 세션 이벤트를 수신** (tmux 스펙)
- `Process` + `Pipe` 로 stdin(명령) / stdout(이벤트) / stderr 연결
- stdout 를 `DispatchSourceRead` 로 비차단 읽기
- **단일 클라이언트 인스턴스** — 앱 생애주기 동안 유지
- 프로세스 종료·연결 끊김 시 exponential backoff(1s→2s→5s→15s cap)로 재연결

**내부 관리 세션 규칙**:
- `_muxbar-` 접두사 세션 (`_muxbar-ctl`, `_muxbar-awake`) 은 **UI 세션 리스트에서 숨김**
- `_muxbar-awake` 만 Keep Awake 토글 인디케이터로 별도 표시

#### 3.1.2 ControlProtocol

tmux control mode 라인 파서. 각 라인의 접두사로 분기:

| 접두사 | 파싱 결과 | 처리 |
|---|---|---|
| `%begin <ts> <cmdId> <flags>` | `CommandBegin(cmdId)` | 응답 버퍼 열기 |
| `%end <ts> <cmdId> <flags>` | `CommandEnd(cmdId, buffer)` | `cmdId` 의 pending Future resolve |
| `%error <ts> <cmdId> <flags>` | `CommandError(cmdId, buffer)` | Future reject |
| `%output %<paneId> <data>` | `PaneOutput(paneId, bytes)` | SessionStore.paneBuffer(paneId).append |
| `%session-changed <id> <name>` | `SessionChanged(...)` | SessionStore 갱신 |
| `%sessions-changed` | `SessionsChanged` | `list-sessions` 재조회 트리거 |
| `%window-add %<id>` / `%window-close %<id>` / `%window-renamed` | `WindowEvent(...)` | 세션 트리 갱신 |
| `%pane-mode-changed %<id>` | `PaneModeChanged(...)` | 프리뷰 모드 감지 |
| `%pause` / `%continue` | flow control | 프리뷰 일시정지 토글 |
| `%exit` | client 종료 통지 | 재연결 루프 진입 |

**이스케이프 처리**: `%output` 의 data 는 `\ooo` (octal) 로 escape 되므로 unescape 필수.

#### 3.1.3 Commands

타입 세이프 래퍼:

```swift
enum TmuxCommand {
  case listSessions
  case listWindows(session: String)
  case killSession(name: String)
  case newSession(name: String, command: String?)
  case capturePane(target: String, lines: Int, escapes: Bool)
  case sendKeys(target: String, keys: [String])
  case renameSession(from: String, to: String)
}

extension ControlClient {
  func send<T: Decodable>(_ cmd: TmuxCommand) async throws -> T
}
```

각 command 는 대기 중인 `cmdId → CheckedContinuation` 맵으로 응답 대기 (Swift Concurrency).

#### 3.1.4 PaneRenderer

- `SwiftTerm` (MIT) 을 의존성으로 추가
- `Terminal` (SwiftTerm 의 headless 엔진) 인스턴스를 pane 당 하나
- `%output` 수신 시 `terminal.feed(bytes)` 호출
- 렌더 시 `terminal.getScrollback()` + `getLines()` → `NSAttributedString`
- SwiftUI 에서는 `NSViewRepresentable` 로 감싼 `NSTextView` 에 표시
- **최적화**: 50 FPS 캡 (20ms throttle). SwiftTerm 의 dirty-line 추적 활용해 변경된 줄만 재렌더

### 3.2 Core Stores

#### 3.2.1 SessionStore

```swift
@MainActor
final class SessionStore: ObservableObject {
  @Published private(set) var sessions: [TmuxSession] = []
  @Published private(set) var connectionState: ConnectionState = .disconnected

  func bootstrap() async  // 초기 list-sessions + 이벤트 구독
  func attach(_ session: TmuxSession, in terminal: TerminalApp) async
  func kill(_ session: TmuxSession) async
  func new(name: String, template: Template?) async
  func preview(for session: TmuxSession) -> PanePreviewHandle
}
```

#### 3.2.2 AwakeStore

```swift
@MainActor
final class AwakeStore: ObservableObject {
  @Published private(set) var isAwake: Bool = false
  @Published private(set) var flags: CaffeinateFlags = .default  // -dims

  func toggle() async throws  // _muxbar-awake 세션 생성/삭제
}
```

**동작**:
- ON: `tmux new-session -d -s _muxbar-awake "caffeinate \(flags.cliArgs); exit"`
  - tmux 서버가 죽으면 caffeinate 도 종료 → Fail-safe
- OFF: `tmux kill-session -t _muxbar-awake`
- 상태 판정: SessionStore 에 `_muxbar-awake` 존재 여부로 판단 (단일 진실 원천)

#### 3.2.3 PreferencesStore

`@AppStorage` 기반. 저장 항목:

| 키 | 타입 | 기본값 |
|---|---|---|
| `launchAtLogin` | Bool | false |
| `preferredTerminal` | String (app bundle id) | 자동 감지 |
| `caffeinateFlags` | String | `dims` |
| `showPreviewOnHover` | Bool | true |
| `previewLines` | Int | 200 |
| `globalHotkeyOpenMenu` | KeyCombo | ⌘⇧T |
| `globalHotkeyAwakeToggle` | KeyCombo | ⌘⇧A |
| `favorites` | [String] | [] (⌘⇧1~9 매핑) |
| `notifyOnIdleMin` | Int? | 30 |
| `notifyOnCrash` | Bool | true |

### 3.3 Features

#### 3.3.1 SessionList UI

메뉴바 드롭다운(MenuBarExtra `.window` 스타일):

```
┌─ muxbar ─────────────────────────────┐
│  [search sessions…]                  │
├──────────────────────────────────────┤
│ ● dev            2w  /msa      attached│
│ ○ api-test       1w  /msa      3m ago │
│ ○ log-tail       1w  /var/log  12m ago│
├──────────────────────────────────────┤
│ ☕ Keep Awake         [   OFF   ]     │
├──────────────────────────────────────┤
│ + New Session  ▸    [Dev][Web][SSH]… │
│ ⚙ Preferences                         │
│ ⏻ Quit                                │
└──────────────────────────────────────┘
```

세션 항목 hover → 우측에 프리뷰 팝오버 (SessionPreview).
세션 항목 클릭 → 컨텍스트 메뉴 (Attach/Preview/Rename/Kill).

#### 3.3.2 SessionPreview

- Trigger: hover(300ms delay) 또는 클릭 → "Preview"
- NSHostingController 로 SwiftTerm 뷰 감싸서 popover
- 초기 `capture-pane -pt {session} -J -e -S -200` 로 히스토리 200줄 로드
- 이후 `%output` 이벤트로 라이브 업데이트
- 크기: 80×24 셀 (설정 가능), 등폭 폰트 SF Mono

#### 3.3.3 Templates

YAML 정의, `Resources/Templates/*.yaml`:

```yaml
name: WebDev
description: editor / dev-server / log-tail 3분할
session_name_hint: web
windows:
  - name: edit
    layout: main-vertical
    panes:
      - cwd: "."
        command: nvim .
      - cwd: "."
        command: npm run dev
  - name: logs
    panes:
      - cwd: "."
        command: tail -f logs/dev.log
```

**기본 제공 템플릿 5종**: Dev, WebDev, Monitoring, SSH, Docker.
사용자 정의 템플릿은 `~/Library/Application Support/muxbar/Templates/` 에서 로드.

#### 3.3.4 TerminalLauncher

Terminal attach 전략:

```swift
enum TerminalApp: String, CaseIterable {
  case terminal     = "com.apple.Terminal"
  case iterm2       = "com.googlecode.iterm2"
  case warp         = "dev.warp.Warp-Stable"
  case alacritty    = "io.alacritty"
  case kitty        = "net.kovidgoyal.kitty"
}

protocol TerminalAdapter {
  func isInstalled() -> Bool
  func attach(sessionName: String) async throws
}
```

구현 세부:

| Terminal | Attach 방법 |
|---|---|
| Terminal.app | `osascript -e 'tell app "Terminal" to do script "tmux a -t \(name)"'` |
| iTerm2 | Python API 또는 AppleScript `tell app "iTerm" to create window with default profile command "tmux a -t \(name)"` |
| Warp | `open -a Warp --args --exec "tmux a -t \(name)"` (Warp가 exec flag 지원 시) / fallback AppleScript |
| Alacritty | `open -na Alacritty --args -e tmux a -t \(name)` |
| Kitty | `open -na kitty --args tmux a -t \(name)` |

자동 감지: `NSWorkspace.shared.urlForApplication(withBundleIdentifier:)` 로 설치 확인. Preferences 에서 기본 앱 지정.

#### 3.3.5 HotKeyCenter

- [`soffes/HotKey`](https://github.com/soffes/HotKey) (MIT) 사용
- 등록: ⌘⇧T(메뉴 토글), ⌘⇧A(Keep Awake), ⌘⇧1~9(즐겨찾기 attach)
- 변경 시 재등록. Preferences UI 는 key combo picker 제공

#### 3.3.6 Notifications

`UserNotifications` 프레임워크:

- **Idle 감지**: `%output` 이벤트를 pane 당 타임스탬프 기록. `notifyOnIdleMin` 경과 시 알림.
- **비정상 종료**: 세션이 사용자 action 없이 사라지면 (`%sessions-changed` 직후 `list-sessions` 비교) 알림.
- 알림 클릭 → 앱 활성화 + 해당 세션으로 스크롤

### 3.4 Login Item

```swift
import ServiceManagement

// Enable
try SMAppService.mainApp.register()
// Disable
try SMAppService.mainApp.unregister()
```

Preferences 에서 체크박스. 상태는 `SMAppService.mainApp.status` 로 동기화.

## 4. UX Details

### 4.1 Menubar Icon States

| 상태 | SF Symbol | Tint |
|---|---|---|
| 기본 (세션 ≥1) | `terminal` | 흰색 (다크모드) / 검정 (라이트) |
| 모든 세션 detached | `terminal` | 50% opacity |
| Attached 세션 있음 | `terminal.fill` + 초록 점 | 시스템 액센트 |
| Awake 활성 | `terminal` + 노란 "Z" 도트 | — |

아이콘은 template 이미지로 지정 (`isTemplate = true`) 하여 시스템이 자동 다크/라이트 대응.

### 4.2 Interaction Shortcuts

- 메뉴 열린 상태에서 **타이핑**으로 필터링 (검색)
- **↑↓** 세션 이동, **Enter** attach (기본 터미널)
- **⌘K** 선택 세션 kill (확인 다이얼로그)
- **⌘⇧K** 확인 없이 kill (위험 action)
- **ESC** 메뉴 닫기

### 4.3 Preferences Tabs

1. **General**: Login Item, 기본 터미널, 메뉴 단축키
2. **Sessions**: 즐겨찾기(⌘⇧1~9) 매핑, 프리뷰 라인 수, hover 지연
3. **Keep Awake**: caffeinate 플래그 체크박스 (d/i/m/s/u), 토글 단축키
4. **Templates**: 리스트/편집/추가
5. **Notifications**: idle 임계값, 비정상 종료 알림 토글
6. **About**: 버전, GitHub 링크, 라이선스, 업데이트 확인 링크

## 5. Data Model

### 5.1 Core Types (Sendable 기반 불변 값 타입)

```swift
struct TmuxSession: Identifiable, Sendable, Equatable {
  let id: String              // tmux session name (unique per server)
  let isAttached: Bool
  let windowCount: Int
  let createdAt: Date
  let lastActivityAt: Date
  let workingDirectory: String?
}

struct TmuxWindow: Identifiable, Sendable, Equatable {
  let id: String              // "@<windowId>" from tmux
  let sessionId: String
  let index: Int
  let name: String
  let paneCount: Int
  let isActive: Bool
}

struct TmuxPane: Identifiable, Sendable, Equatable {
  let id: String              // "%<paneId>"
  let windowId: String
  let command: String?
  let pid: pid_t?
  let isActive: Bool
}

struct CaffeinateFlags: Sendable, Equatable {
  var preventDisplaySleep: Bool     // -d
  var preventIdleSleep: Bool        // -i
  var preventDiskIdleSleep: Bool    // -m
  var preventSystemSleep: Bool      // -s
  var preventUserIdleSleep: Bool    // -u

  static let `default` = CaffeinateFlags(d: true, i: true, m: true, s: true, u: false)
  var cliArgs: String  // "-dims"
}

struct Template: Codable, Identifiable, Sendable {
  let id: UUID
  var name: String
  var description: String
  var sessionNameHint: String
  var windows: [TemplateWindow]
}
```

### 5.2 ConnectionState

```swift
enum ConnectionState: Sendable {
  case disconnected
  case connecting(attempt: Int)
  case connected
  case reconnecting(nextAttemptIn: TimeInterval)
  case failed(reason: String)  // tmux 바이너리 없음 등 복구 불능
}
```

## 6. Error Handling & Edge Cases

| 케이스 | 처리 |
|---|---|
| `tmux` 바이너리 미설치 | 시작 시 검출. 메뉴바 아이콘 경고, "Install via Homebrew" 안내. `brew list tmux` 로 확인. |
| tmux 서버 죽음 (`%exit`) | ControlClient 재연결 루프, 사용자에게 토스트 알림 |
| 네트워크/마운트 끊김으로 cwd 해석 실패 | `?` 로 표시, 폴백 없이 계속 |
| 세션에 한글/공백/특수문자 | `list-sessions -F "#{session_name}"` 결과 그대로 사용, attach 시 shell-quote 처리 |
| 같은 세션을 두 터미널에서 attach | tmux 자체가 허용 (다중 클라이언트), muxbar는 상태만 갱신 |
| `_muxbar-awake` 을 사용자가 직접 kill | SessionStore 가 `%sessions-changed` 수신 → AwakeStore.isAwake=false 로 동기화 |
| `_muxbar-ctl` 을 사용자가 실수로 kill | ControlClient 가 `%exit` 수신 → 재부트스트랩 시퀀스 재실행 |
| caffeinate 플래그가 유효하지 않음 | 실행 전 validator, 최소 1개 flag 필수 |
| Homebrew 경로 다름 (`/opt/homebrew/bin/tmux` vs `/usr/local/bin/tmux`) | `PATH` 에서 동적 해석, 실패 시 일반 경로 fallback |
| Apple Silicon / Intel | 양쪽 빌드 후 universal binary 로 배포 (`-arch arm64 -arch x86_64`) |
| 재부팅 직후 tmux 서버 없음 | bootstrap 이 자동으로 새 서버 기동 (`tmux -CC new-session`) |

### 6.1 Crash Safety

- tmux Process 크래시는 앱 크래시로 이어지지 않아야 함 → `NotificationCenter` 로 terminate 감지, 재연결
- 모든 async task 는 `Task { }` 안에서 `do/catch` 필수
- 치명적 에러는 `OSLog.error` 로 기록

## 7. Testing Strategy

### 7.1 단위 테스트 (TmuxKitTests)

- `ControlProtocol` 파서: 각 `%` 이벤트 fixture 로 검증
- octal escape unescape 코너 케이스
- `CaffeinateFlags.cliArgs` 조합 테스트

### 7.2 통합 테스트 (IntegrationTests)

- 실제 `tmux` 바이너리 실행 요구 (CI에 tmux 설치)
- 시나리오:
  - 빈 서버 → `new-session` → `list-sessions` 결과 검증
  - `%output` 이벤트 스트리밍 → SwiftTerm 렌더 결과 검증
  - `kill-session` → `%sessions-changed` 수신

### 7.3 UI 테스트 (수동 체크리스트)

- 초기 MVP 는 UI 자동화 생략, 체크리스트로 대체
- 핵심 시나리오:
  - [ ] 메뉴 열기 → 세션 리스트 표시
  - [ ] Attach → 선택 터미널에서 열림
  - [ ] Kill → 확인 다이얼로그 → 세션 사라짐
  - [ ] Keep Awake 토글 → `pmset -g assertions` 로 확인
  - [ ] Login Item 체크 → 재부팅 후 자동 실행

### 7.4 CI

- GitHub Actions macOS-14 runner
- `swift build` + `swift test` + SwiftLint
- 릴리스 태그 푸시 시 `Distribution/Release.sh` 실행, `.dmg` 업로드

## 8. Distribution

### 8.1 Build

```bash
# Release.sh
swift build -c release --arch arm64 --arch x86_64
create-dmg \
  --volname "muxbar \(version)" \
  --app-drop-link 450 120 \
  dist/muxbar-\(version).dmg \
  .build/apple/Products/Release/muxbar.app
# Ad-hoc sign (개발자 계정 없음)
codesign --deep --force --sign - dist/muxbar-\(version).dmg
```

### 8.2 GitHub Releases

- Tag: `v0.1.0`
- Asset: `muxbar-0.1.0.dmg`
- Release notes 자동 생성 (git-cliff 또는 release-drafter)

### 8.3 Homebrew Tap

**레포**: `github.com/1989v/homebrew-tap` (별도)

**cask 정의** (`Casks/muxbar.rb`):

```ruby
cask "muxbar" do
  version "0.1.0"
  sha256 "..."

  url "https://github.com/1989v/muxbar/releases/download/v#{version}/muxbar-#{version}.dmg"
  name "muxbar"
  desc "tmux session manager with caffeinate toggle in the menu bar"
  homepage "https://github.com/1989v/muxbar"

  depends_on formula: "tmux"
  depends_on macos: ">= :ventura"

  app "muxbar.app"

  postflight do
    system_command "xattr",
                   args: ["-dr", "com.apple.quarantine", "#{appdir}/muxbar.app"],
                   sudo: false
  end

  zap trash: [
    "~/Library/Application Support/muxbar",
    "~/Library/Preferences/com.1989v.muxbar.plist",
  ]
end
```

설치: `brew install --cask 1989v/tap/muxbar`

### 8.4 First-Launch UX

- README 에 명시: "처음 실행 시 우클릭 → 열기"
- Homebrew cask 설치 시에는 postflight 가 quarantine 제거해 즉시 실행 가능
