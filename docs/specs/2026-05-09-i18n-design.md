# muxbar — i18n (English + 한국어) Design

- **Date**: 2026-05-09
- **Status**: Draft (awaiting user review)
- **Author**: kgd
- **Related**: `docs/specs/2026-04-17-v0.1-design.md`, `docs/specs/2026-05-08-closed-lid-mode-design.md`

---

## 1. Overview

muxbar 의 모든 user-facing 텍스트를 영어 + 한국어로 제공. 첫 실행 시 macOS system locale 자동 감지, Settings 에서 사용자가 수동 override 가능 (Auto / English / 한국어).

### 1.1 Problem

- 현재 모든 메뉴/버튼/에러 메시지가 영어로 하드코딩 (~48개 string)
- 한국어 사용자에게 진입 장벽: 메뉴바 한 줄짜리 라벨조차 영어
- 한국어 시장이 muxbar 의 1차 타겟 audience 중 하나 (`docs/guides/2026-05-09-launch-plan.md` 의 GeekNews/OKKY 채널)

### 1.2 Goals

| # | Goal | 측정 |
|---|---|---|
| G1 | macOS system locale 자동 감지 | 한국어 macOS 에서 첫 실행 → 메뉴 한국어 |
| G2 | Settings 에서 수동 override (Auto / English / 한국어) | 3 옵션 picker 동작 |
| G3 | 언어 변경 시 매끄러운 재시작 | Alert 확인 → auto quit + relaunch → 새 언어 적용 |
| G4 | i18n 인프라 확장 가능 | 추가 언어는 `<lang>.lproj` 디렉토리만 추가하면 자동 활성 |
| G5 | 누락된 번역 fallback | ko 에 key 없으면 en, en 에도 없으면 key 자체 표시 (개발 중 발견) |

### 1.3 Non-Goals

- 일본어/중국어 등 추가 언어 (이번 spec 은 en/ko 만)
- README.md / README.ko.md 의 i18n (이미 별 파일로 운영, 변경 없음)
- Plurals 처리 (현재 string 에 plural 거의 없음)
- RTL 언어 지원

## 2. Architecture

```
┌──────────────────────────────────────────────────────┐
│ Sources/Core/                                        │
│   Resources/                                         │
│     en.lproj/Localizable.strings   ← base + 영어     │
│     ko.lproj/Localizable.strings   ← 한국어 번역      │
│   I18n.swift                                         │
│     enum L { static let key1: String; ... }          │ ← 타입세이프 키
│     enum LanguagePreference { auto, en, ko }         │ ← Settings 모델
│     final class LocaleService                        │ ← 저장 + 적용 + 재시작
├──────────────────────────────────────────────────────┤
│ Package.swift                                        │
│   Core target 에 resources: [.process("Resources")] │
├──────────────────────────────────────────────────────┤
│ Sources/Features/Settings/SettingsMenu.swift         │
│   Language picker (Auto / English / 한국어)          │
├──────────────────────────────────────────────────────┤
│ 모든 SwiftUI View 에서:                              │
│   Text(L.menuKeepAwake)                              │
│   Button(L.attach) { ... }                           │
└──────────────────────────────────────────────────────┘
```

핵심 결정:
- 모든 user-facing string 을 **`Core` 모듈 한 곳에 통합** (모듈별 분산 X). 작은 앱(48 string) 이라 단일 source of truth 가 단순.
- 모든 lookup 은 `Bundle.module` (Core 의 SPM 자동 생성 bundle) 기반.

### 3.2 `Sources/Core/I18n.swift`

```swift
import Foundation

/// 타입세이프 string key — 오타 방지 + IDE 자동완성.
public enum L {
    public static let menuKeepAwake = key("menu.keepAwake")
    public static let menuClosedLid = key("menu.closedLid")
    // ... ~50 keys
    
    private static func key(_ k: String) -> String {
        NSLocalizedString(k, bundle: .module, comment: "")
    }
    
    /// %@ 가 있는 format key 는 String(format:) 으로 호출.
    public static func closedLidStateOnTimer(_ time: String) -> String {
        String(format: NSLocalizedString("closedLid.state.onTimer", bundle: .module, comment: ""), time)
    }
}

/// Settings 모델.
public enum LanguagePreference: String, Codable, CaseIterable, Identifiable {
    case auto, en, ko
    public var id: String { rawValue }
}

/// 언어 preference 저장 + 적용 + 재시작.
@MainActor
public final class LocaleService: ObservableObject {
    public static let key = "muxbar.language"

    @Published public var preference: LanguagePreference {
        didSet { defaults.set(preference.rawValue, forKey: Self.key) }
    }
    private let defaults: UserDefaults

    public init(defaults: UserDefaults = .standard) {
        self.defaults = defaults
        let raw = defaults.string(forKey: Self.key) ?? LanguagePreference.auto.rawValue
        self.preference = LanguagePreference(rawValue: raw) ?? .auto
    }

    /// 앱 시작 시 호출 — preference 따라 AppleLanguages 결정.
    /// .auto 면 시스템 그대로, 그 외엔 forced override.
    public func applyAtLaunch() {
        switch preference {
        case .auto:
            // AppleLanguages 명시적 set 안 함 — macOS 가 시스템 locale 따름
            defaults.removeObject(forKey: "AppleLanguages")
        case .en:
            defaults.set(["en"], forKey: "AppleLanguages")
        case .ko:
            defaults.set(["ko"], forKey: "AppleLanguages")
        }
    }

    /// Settings 에서 변경 후 호출 — preference 저장 + 재시작.
    public func applyAndRelaunch() {
        applyAtLaunch()
        let url = Bundle.main.bundleURL
        let config = NSWorkspace.OpenConfiguration()
        config.createsNewApplicationInstance = true
        NSWorkspace.shared.openApplication(at: url, configuration: config) { _, _ in
            DispatchQueue.main.async { NSApp.terminate(nil) }
        }
    }
}
```

`Bundle.module` 은 SPM 이 target resources 처리 시 자동 생성하는 bundle. `Core` target 에 resources 등록되면 `Bundle.module` accessor 가 자동 노출됨.

### 3.3 `Package.swift` 변경

`Core` target 정의에 resources 추가:

```swift
.target(
    name: "Core",
    dependencies: ["MuxLogging", .product(name: "Logging", package: "swift-log")],
    resources: [
        .process("Resources"),
    ],
    swiftSettings: [.enableExperimentalFeature("StrictConcurrency")]
),
```

`.process(...)` 는 SPM 이 `.lproj` 디렉토리를 자동 인식 + main bundle 에 패키징.

### 3.4 `SettingsMenu.swift` 확장

기존 Open at Login + Closed-lid 토글 다음 Divider + Language picker:

```swift
Divider()
Text(L.settingsLanguage).font(.caption).foregroundStyle(.secondary)
Picker(selection: $localeService.preference) {
    Text(L.settingsLanguageAuto).tag(LanguagePreference.auto)
    Text(L.settingsLanguageEn).tag(LanguagePreference.en)
    Text(L.settingsLanguageKo).tag(LanguagePreference.ko)
} label: {
    Text(L.settingsLanguage)
}
.onChange(of: localeService.preference) { _, _ in
    showRelaunchAlert = true
}
.alert(L.settingsRelaunchTitle, isPresented: $showRelaunchAlert) {
    Button(L.settingsRelaunchConfirm) { localeService.applyAndRelaunch() }
    Button(L.settingsRelaunchCancel, role: .cancel) {}
} message: {
    Text(L.settingsRelaunchBody)
}
```

`@ObservedObject var localeService: LocaleService` 가 init parameter 추가.

### 3.5 `AppState.swift` + `MuxBarApp.swift` wiring

`AppState` 에 `localeService: LocaleService` 추가, init 에서 instantiate. `MuxBarApp` 의 init 에서 `appState.localeService.applyAtLaunch()` 호출 (가장 이른 시점에 AppleLanguages 결정).

`SettingsMenu(...)` 호출에 `localeService:` 추가.

## 4. Data Flow — 언어 변경

```
1. 사용자가 Settings → Language → "한국어" 선택
2. Picker binding → LocaleService.preference = .ko
3. didSet → UserDefaults "muxbar.language" = "ko"
4. SwiftUI .onChange → showRelaunchAlert = true
5. Alert 표시 → 사용자 "Restart" 클릭
6. localeService.applyAndRelaunch():
   - UserDefaults "AppleLanguages" = ["ko"]
   - NSWorkspace.openApplication(at: bundleURL, createsNewApplicationInstance: true)
   - NSApp.terminate(nil)
7. 새 muxbar 인스턴스 시작 → init 에서 applyAtLaunch() (이미 set 돼있음, no-op)
8. SwiftUI Text(L.x) 가 ko.lproj lookup → 한국어 표시
```

`.auto` 로 돌아갈 때:
```
LocaleService.preference = .auto
→ UserDefaults "muxbar.language" = "auto"
→ Restart 시 applyAtLaunch() 가 AppleLanguages 키 제거
→ macOS 가 시스템 locale 따름
```

## 5. Error Handling

| 시나리오 | 처리 |
|---|---|
| 키가 어느 lproj 에도 없음 | NSLocalizedString 이 key 자체 반환 (예: `"menu.keepAwake"`) — 개발 중 즉시 발견 |
| ko.lproj 에 누락 | en.lproj fallback (Apple 내장 동작) |
| AppleLanguages override 실패 | UserDefaults set 실패는 거의 없음. 실패해도 .auto 와 동일 동작 |
| relaunch 실패 (NSWorkspace.openApplication 콜백 error) | log 만 — 사용자가 수동 재시작 |
| Bundle.module 미생성 (Package.swift resources 누락) | 컴파일 시점에 SPM 이 generate 안 함 → 빌드 에러 → 즉시 발견 |

## 6. State Indication

언어 picker 의 현재 선택 → radio dot.
변경 직후 → relaunch alert (modal).
재시작 후 → 새 언어로 모든 라벨 즉시 표시.

언어 picker 자체의 라벨 ("Auto", "English", "한국어") 은 **각 lproj 에서 native name** 으로 표기 — Auto 는 그대로 "Auto", English 는 "English" / "영어" 가 아닌 "English" 그대로 유지 (양 lproj 모두 "English" 와 "한국어" native).

## 7. Testing

### 7.1 Unit

`Tests/CoreTests/I18nTests.swift`:

- `test_allKeys_existInBothLprojs` — en.lproj 의 모든 key 가 ko.lproj 에도 있는지 (양쪽 .strings 파일 read + key set 비교).
- `test_languagePreference_defaultsToAuto` — UserDefaults 비어있을 때 `.auto`.
- `test_languagePreference_persists` — preference 변경 시 UserDefaults 저장 + 재로드 시 복원.
- `test_applyAtLaunch_auto_removesAppleLanguages` — `.auto` 면 AppleLanguages key 제거.
- `test_applyAtLaunch_en_setsEnglishOverride` — `.en` 이면 AppleLanguages = ["en"].
- `test_applyAtLaunch_ko_setsKoreanOverride` — `.ko` 이면 AppleLanguages = ["ko"].

### 7.2 Manual

- macOS 시스템 한국어 설정 → 첫 실행 → 메뉴 한국어
- 시스템 영어 설정 → 첫 실행 → 메뉴 영어
- Settings → Language → English 선택 → Alert → Restart → 영어 적용
- Settings → Language → 한국어 선택 → Alert → Cancel → 변경 취소 (relaunch 안 함)
- 모든 user-facing string 시각 검수 (메뉴, popover, alert 등)

## 8. Out of Scope

- 추가 언어 (일본어/중국어 등) — 이번 spec 은 en/ko 만. 추가는 `<lang>.lproj` 디렉토리 추가만.
- README.md / README.ko.md 의 i18n — 별 파일 운영
- Plurals (`%lld items`) — 현재 string 거의 없음
- RTL 언어
- 동적 한국어 변환 도구 (자동 번역) — 모든 번역은 hand-curated
- Localizable.strings 의 변수 보간 외 복잡 format — 현재는 closed-lid countdown `%@` 한 건만

## 9. Open Questions / Risks

- **Bundle.module 의 SPM `swift build` 호환성**: Swift 5.9+ 에서 `.process("Resources")` 는 안정적으로 .lproj 인식. 단, build script 가 환경 따라 차이 가능 — Task 1 빌드 검증으로 확인.
- **AppleLanguages override 의 부수효과**: 시스템 dialog (예: AppleScript admin prompt) 가 muxbar 의 override locale 따름 (single-app domain UserDefaults 라 영향 X). 단 macOS 일부 system service 는 시스템 locale 우선 — closed-lid mode 의 admin prompt 는 시스템 locale.
- **언어 변경 시점 quit/relaunch** 가 사용자에게 충격적이지 않게: alert body 에 명확한 안내 + Cancel 옵션.
